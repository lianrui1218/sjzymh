# 易普力数据资源门户研发排期计划

---

## 一、项目概述

### 1.1 项目背景

易普力数据资源门户是企业级数据资源统一管理平台，旨在实现数据资源统一汇聚、数据权限精准管控、数据质量持续监控和数据价值高效释放。项目基于已完成的PRD需求文档进行研发排期，以4月底为最终交付节点。

### 1.2 项目目标

| 阶段 | 目标 |
|------|------|
| V1.0 | 完成核心功能开发，实现基本业务闭环 |
| 上线目标 | 4月底具备上线条件，系统稳定运行 |

### 1.3 项目范围

**包含范围**：
- 首页模块（Banner、全局搜索、我的待办、数据概览、资源推荐等）
- 数据资源模块（资源浏览、详情、申请、收藏）
- 数据质量模块（六维质量评估、部门质量概览）
- 通知公告模块（公告浏览、详情）
- 管理后台模块（公告管理、规范管理）
- 泛微OA集成（统一登录、消息推送）

**不包含范围**：
- 移动端应用（后续版本考虑）
- 数据可视化大屏（后续版本考虑）
- 高级数据分析功能（后续版本考虑）

## 五、泛微OA对接方案

### 5.1 统一登录（SSO）

#### 5.1.1 对接架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   泛微OA门户     │────▶│   数据门户前端    │────▶│   数据门户后端   │
│   (IdP)         │◀────│   (SP)          │◀────│   (验证服务)     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

#### 5.1.2 技术方案

| 项目 | 方案说明 |
|------|----------|
| 认证协议 | SAML 2.0 / OAuth 2.0 |
| 实现方式 | 泛微OA作为IdP，数据门户作为SP |
| Token机制 | JWT Token，有效期24小时 |
| 登录流程 | 1. 用户点击OA门户入口<br>2. 携带SAML Response/Authorization Code<br>3. 数据门户验证Token<br>4. 登录成功，创建会话 |

#### 5.1.3 接口规范

**用户信息同步接口**：

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | String | 用户ID（泛微OA工号） |
| userName | String | 用户姓名 |
| department | String | 所属部门 |
| role | String | 用户角色 |
| email | String | 邮箱 |
| mobile | String | 手机号 |

#### 5.1.4 开发任务

| 任务 | 预计工时 | 负责人 |
|------|----------|--------|
| SSO配置 | 8h | 架构师 |
| 后端登录验证 | 16h | 后端开发 |
| 前端登录处理 | 8h | 前端开发 |
| 用户同步定时任务 | 8h | 后端开发 |
| 登录测试 | 8h | 测试 |

### 5.2 消息推送

#### 5.2.1 对接架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   数据门户后端   │────▶│   泛微OA接口    │────▶│   泛微OA待办    │
│   (消息触发)    │     │   (推送服务)    │     │   (待办中心)    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

#### 5.2.2 技术方案

| 项目 | 方案说明 |
|------|----------|
| 推送方式 | 泛微OA开放平台API（E9接口） |
| 消息类型 | 待办事项、审批提醒 |
| 触发场景 | 权限申请提交、审批节点变化 |
| 回调处理 | 点击待办跳转回数据门户对应页面 |

#### 5.2.3 接口规范

**推送待办接口**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| taskId | String | 是 | 待办ID |
| title | String | 是 | 待办标题 |
| content | String | 是 | 待办内容 |
| receiver | String | 是 | 接收人工号 |
| creator | String | 是 | 创建人工号 |
| url | String | 否 | 点击跳转URL |
| deadline | Date | 否 | 截止时间 |

| 任务 | 预计工时 | 负责人 |

| 泛微OA API调研 | 8h | 架构师 |

| 消息模板配置 | 8h | 后端开发 |

## 六、风险管控

### 6.1 风险识别

| 风险编号 | 风险描述 | 风险等级 | 应对措施 |
|----------|----------|----------|----------|
| R1 | 泛微OA接口不稳定影响集成 | 高 | 提前进行接口联调，预留缓冲时间 |
| R2 | 需求变更影响进度 | 中 | 建立变更控制流程，优先保证核心功能 |
| R3 | 测试时间不足 | 中 | 增加测试资源，优化测试用例 |
| R4 | 人员变动影响 | 低 | 交叉培训，关键岗位有备份 |
| R5 | 第三方服务故障 | 低 | 做好降级方案 |

### 6.2 应急预案

| 场景 | 应急方案 |
|------|----------|
| 泛微OA集成延期 | 先实现本地登录，集成作为二期优化 |
| 测试发现严重Bug | 优先修复，调整上线时间点 |
| 性能不达标 | 降级非核心功能，确保主流程可用 |
