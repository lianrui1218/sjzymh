# Tasks

本文档为研发团队提供登录状态与未登录状态数据展示与流程状态梳理的任务清单。

## 任务清单

- [ ] Task 1: 前端登录状态管理实现
  - [ ] SubTask 1.1: 实现全局登录状态管理（使用Vuex/Redux或Context API）
  - [ ] SubTask 1.2: 实现Token存储与验证逻辑（Cookie或LocalStorage）
  - [ ] SubTask 1.3: 实现登录状态判断函数（检查Token是否存在且有效）
  - [ ] SubTask 1.4: 实现登录状态变化时的页面刷新机制

- [ ] Task 2: 导航栏状态展示实现
  - [ ] SubTask 2.1: 实现已登录状态导航栏（用户头像、用户名、通知图标、用户菜单）
  - [ ] SubTask 2.2: 实现未登录状态导航栏（登录按钮）
  - [ ] SubTask 2.3: 实现用户菜单下拉框（个人中心、我的申请、我的资源、修改密码、管理后台、退出登录）
  - [ ] SubTask 2.4: 实现通知图标未读消息数量角标

- [ ] Task 3: 首页我的待办模块状态展示
  - [ ] SubTask 3.1: 实现已登录状态的完整待办模块（四个标签页及数据）
  - [ ] SubTask 3.2: 实现未登录状态的登录引导界面
  - [ ] SubTask 3.3: 实现登录引导界面的登录按钮和功能说明

- [ ] Task 4: 数据资源页面状态展示
  - [ ] SubTask 4.1: 实现已登录状态的操作按钮（收藏、申请资源）
  - [ ] SubTask 4.2: 实现未登录状态的操作按钮（登录后申请、登录后收藏）
  - [ ] SubTask 4.3: 实现点击未登录状态按钮的登录提示框

- [ ] Task 5: 资源详情页状态展示
  - [ ] SubTask 5.1: 实现已登录状态的完整详情页（基础信息、字段信息、权限信息）
  - [ ] SubTask 5.2: 实现未登录状态的部分详情页（仅基础信息）
  - [ ] SubTask 5.3: 实现敏感数据隐藏逻辑（字段信息、权限信息）

- [ ] Task 6: 登录提示框组件实现
  - [ ] SubTask 6.1: 创建统一的登录提示框组件
  - [ ] SubTask 6.2: 实现提示框内容（标题、提示文字、登录按钮、取消按钮、功能说明）
  - [ ] SubTask 6.3: 实现提示框的调用逻辑（在需要登录的功能触发时显示）

- [ ] Task 7: 路由守卫实现
  - [ ] SubTask 7.1: 实现需要登录页面的路由守卫（apply.html、admin.html）
  - [ ] SubTask 7.2: 实现管理员页面的权限验证
  - [ ] SubTask 7.3: 实现未授权访问的提示和重定向

- [ ] Task 8: 后端接口设计
  - [ ] SubTask 8.1: 设计公开数据接口（无需Token验证）
  - [ ] SubTask 8.2: 设计登录数据接口（需要Token验证）
  - [ ] SubTask 8.3: 设计敏感数据接口（需要Token验证 + 权限验证）
  - [ ] SubTask 8.4: 实现统一的错误码返回格式

- [ ] Task 9: 泛微OA集成实现
  - [ ] SubTask 9.1: 实现泛微OA登录跳转逻辑（携带回调URL）
  - [ ] SubTask 9.2: 实现Token验证和用户信息获取
  - [ ] SubTask 9.3: 实现退出登录逻辑（清除Token、调用泛微OA登出接口）

- [ ] Task 10: 登录过期处理
  - [ ] SubTask 10.1: 实现Token过期检测逻辑
  - [ ] SubTask 10.2: 实现登录过期提示框
  - [ ] SubTask 10.3: 实现用户操作上下文保存与恢复

## 任务依赖关系

- Task 1 是所有任务的基础，必须优先完成
- Task 2、Task 3、Task 4、Task 5 依赖于 Task 1
- Task 6 可以与 Task 2-5 并行开发
- Task 7 依赖于 Task 1 和 Task 6
- Task 8 可以与前端任务并行开发
- Task 9 依赖于 Task 1 和 Task 8
- Task 10 依赖于 Task 1 和 Task 9

## 开发优先级

### 高优先级（P0）
- Task 1: 前端登录状态管理实现
- Task 2: 导航栏状态展示实现
- Task 8: 后端接口设计
- Task 9: 泛微OA集成实现

### 中优先级（P1）
- Task 3: 首页我的待办模块状态展示
- Task 4: 数据资源页面状态展示
- Task 6: 登录提示框组件实现
- Task 7: 路由守卫实现

### 低优先级（P2）
- Task 5: 资源详情页状态展示
- Task 10: 登录过期处理

## 验收标准

- 所有页面在已登录和未登录状态下展示正确的内容
- 所有需要登录的功能在未登录状态下有清晰的登录引导
- 登录流程顺畅，用户体验良好
- 后端接口返回正确的数据和错误码
- 泛微OA集成正常，登录和退出功能正常
