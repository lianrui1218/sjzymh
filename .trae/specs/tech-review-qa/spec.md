# 技术负责人疑问与产品负责人解答 Spec

## 一、技术负责人提出的疑问

### 1. 技术架构相关疑问

| 编号 | 疑问内容 | 影响范围 |
|------|---------|---------|
| T1 | 用户认证方式未明确：是否对接企业SSO/AD域？还是独立账号体系？ | 用户管理、登录流程 |
| T2 | 前端技术栈未明确：是否使用现有技术栈？是否需要服务端渲染(SSR)？ | 前端开发、部署方式 |
| T3 | 后端服务架构：是单体应用还是微服务架构？是否需要拆分服务？ | 后端开发、部署运维 |
| T4 | 数据库选型：关系型数据库(MySQL/PostgreSQL)还是需要混合存储？ | 数据存储、性能 |

### 2. 数据接口相关疑问

| 编号 | 疑问内容 | 影响范围 |
|------|---------|---------|
| D1 | 元数据管理系统接口：是否已有现成接口？接口格式是什么？ | 数据表总量统计 |
| D2 | API网关系统接口：如何获取API接口统计数据？ | API接口总量统计 |
| D3 | 数据质量检测系统接口：质量数据如何获取？推送还是拉取？ | 数据质量模块 |
| D4 | 审批流程系统：是否对接OA系统？还是自建审批流程？ | 权限申请、审批流程 |
| D5 | 用户组织架构数据：从哪个系统获取？HR系统还是AD域？ | 用户管理、部门数据 |

### 3. 性能相关疑问

| 编号 | 疑问内容 | 影响范围 |
|------|---------|---------|
| P1 | 资源列表分页：默认每页显示多少条？是否支持前端分页？ | 资源推荐模块 |
| P2 | 数据质量检测频率：每日检测的具体时间？检测是否影响业务系统？ | 数据质量模块 |
| P3 | 埋点数据存储：埋点数据量预估多大？存储方案是什么？ | 运营埋点需求 |
| P4 | 审计日志存储：日志量预估多大？是否需要日志归档策略？ | 审计日志需求 |

### 4. 安全相关疑问

| 编号 | 疑问内容 | 影响范围 |
|------|---------|---------|
| S1 | 敏感数据展示：资源详情中的数据是否需要脱敏？脱敏规则是什么？ | 资源详情弹窗 |
| S2 | API调用安全：如果提供API预览功能，如何保证安全？ | 资源详情弹窗 |
| S3 | 文件下载安全：标准规范文档下载是否需要水印？ | 标准规范模块 |
| S4 | 会话管理：登录会话超时时间是多少？是否支持单点登录？ | 用户认证 |

### 5. 可扩展性相关疑问

| 编号 | 疑问内容 | 影响范围 |
|------|---------|---------|
| E1 | 资源类型扩展：未来是否需要支持新的资源类型（如数据服务、数据应用）？ | 资源管理 |
| E2 | 审批流程扩展：是否需要支持多级审批？会签？ | 审批流程 |
| E3 | 数据质量维度扩展：是否需要支持自定义质量维度？ | 数据质量模块 |
| E4 | 分类体系扩展：业务域分类是否需要支持多级分类？ | 资源分类 |

### 6. 运维相关疑问

| 编号 | 疑问内容 | 影响范围 |
|------|---------|---------|
| O1 | 部署环境：是私有化部署还是云部署？服务器资源预估？ | 系统部署 |
| O2 | 监控告警：是否需要系统监控？告警通知方式？ | 系统运维 |
| O3 | 数据备份：数据备份策略是什么？恢复时间目标(RTO)？ | 数据安全 |
| O4 | 日志管理：应用日志如何收集和分析？ | 问题排查 |

### 7. 业务逻辑相关疑问

| 编号 | 疑问内容 | 影响范围 |
|------|---------|---------|
| B1 | 审批人分配规则：系统如何自动分配审批人？按部门？按资源？ | 权限申请流程 |
| B2 | 资源责任人管理：责任人如何指定？是否支持多人负责？ | 资源管理 |
| B3 | 部门数据归属：一个数据表可能被多个部门使用，如何归属？ | 数据质量模块 |
| B4 | 权限回收机制：用户离职时权限如何处理？ | 权限管理 |
