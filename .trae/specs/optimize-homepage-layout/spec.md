# 首页HTML布局优化 Spec

## Why
首页顶部搜索框宽度过大，搜索提示词不够简洁，核心数据概览模块的标题和更多按钮与下方卡片边框未对齐，影响页面整体视觉效果和用户体验。

## What Changes
- 顶部搜索框宽度减少一半（从70%减少到35%）
- 搜索提示词简化为"搜索资源名称"
- 核心数据概览标题与下方卡片边框左对齐
- 核心数据概览标题高度与"我的待办"标题一致
- 核心数据概览模块右边"更多"按钮与下方卡片边框右对齐

## Impact
- Affected code: index.html

## ADDED Requirements

### Requirement: 搜索框宽度优化
系统应将顶部搜索框的最大宽度从70%减少到35%，使搜索框更加紧凑美观。

#### Scenario: 搜索框宽度调整
- **WHEN** 用户访问首页
- **THEN** 顶部搜索框宽度应为原来的50%（35%）

### Requirement: 搜索提示词简化
系统应将搜索框的placeholder文本简化为"搜索资源名称"。

#### Scenario: 搜索提示词显示
- **WHEN** 用户查看搜索框
- **THEN** placeholder显示"搜索资源名称"

### Requirement: 核心数据概览标题对齐
系统应确保核心数据概览标题与下方卡片边框左对齐，且标题高度与"我的待办"标题一致。

#### Scenario: 标题左对齐
- **WHEN** 用户查看核心数据概览模块
- **THEN** 标题与下方卡片左边框对齐

#### Scenario: 标题高度一致
- **WHEN** 用户同时查看"我的待办"和"核心数据概览"模块
- **THEN** 两个模块的标题高度一致

### Requirement: 更多按钮右对齐
系统应确保核心数据概览模块右边的"更多"按钮与下方卡片右边框对齐。

#### Scenario: 更多按钮对齐
- **WHEN** 用户查看核心数据概览模块
- **THEN** "更多"按钮与下方卡片右边框对齐
