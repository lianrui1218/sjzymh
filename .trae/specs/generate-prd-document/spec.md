# 易普力数据资源门户 产品需求文档（PRD）

## 一、文档说明

### 1. 文档用途
本产品需求说明书（PRD）用于项目立项、开发、测试、验收。

### 2. 文档来源
基于现有易普力数据资源门户HTML页面反向梳理。

### 3. 适用对象
产品、研发、测试、项目经理、领导

---

## 二、产品整体定位

### 1. 产品核心目标
构建企业级数据资源统一管理门户，实现数据资产的集中管理、统一授权、质量监控和价值挖掘，支撑企业数字化转型和数据驱动决策。

### 2. 面向用户/角色
| 角色 | 职责描述 |
|------|--------|
| 普通员工 | 申请数据资源访问权限、查看报表、收藏常用资源 |
| 部门主管 | 审批数据权限申请、管理本部门数据资源 |
| 数据管理员 | 管理数据质量、维护数据标准规范、配置权限策略 |
| 系统管理员 | 系统配置、用户管理、日志审计 |

### 3. 核心价值
- **数据资产可视化**：统一展示企业数据表、API、指标、报表等资源
- **权限管控规范化**：标准化数据访问申请与审批流程
- **数据质量可量化**：多维度监控数据质量达标情况
- **提升数据利用效率**：推荐机制帮助用户快速找到所需资源

## 四、详细功能需求

### M1：Banner轮播模块

#### 1. 模块定义
首页顶部全屏轮播展示区，用于展示企业宣传标语、核心价值主张和全局搜索入口。

#### 2. 页面入口与路径
- **入口**：首页顶部
- **路径**：用户访问首页自动展示

#### 3. 核心交互流程
1. 页面加载后自动开始轮播
2. 用户可点击底部圆点切换轮播图
3. 用户可在搜索框输入关键词搜索资源
4. 可点击搜索框下方标签快速筛选

#### 4. 字段/组件说明
| 组件名称 | 类型 | 说明 |
|---------|------|------|
| 轮播图 | 图片/背景 | 4张轮播图，自动切换间隔5秒 |
| 主标题 | 文本 | "数据驱动价值 创新引领未来" |
| 副标题 | 文本 | "统一数据资产管理，赋能企业数字化转型" |
| 搜索框 | 输入框 | 支持关键词搜索，最大长度100字符 |
| 筛选标签 | 按钮 | "数据表"、"API"、"报表"、"指标"四个快捷筛选标签 |

#### 5. 业务规则与约束
- 轮播图自动切换间隔：5秒
- 搜索支持模糊匹配资源名称、描述
- 点击筛选标签后跳转至资源列表页并自动筛选

#### 6. 异常/边界场景
- 轮播图加载失败显示默认背景
- 搜索无结果时显示"未找到相关资源"

### M2：我的待办模块

#### 1. 模块定义
展示当前用户相关的待办事项、申请记录、收藏资源和已办结事项，支持Tab切换查看不同类型内容。

#### 2. 页面入口与路径
- **入口**：首页三列布局左侧
- **路径**：首页 → 我的待办模块

#### 3. 核心交互流程
1. 用户查看待办列表
2. 点击Tab切换不同类型（我的待办/我的申请/我的收藏/已办结）
3. 点击待办项跳转至审批详情页
4. 点击申请项查看申请进度

#### 4. 字段/组件说明
| 字段名称 | 类型 | 说明 |
|---------|------|------|
| Tab标签 | 导航 | 我的待办/我的申请/我的收藏/已办结 |
| 待办标题 | 文本 | 审批事项名称 |
| 申请人 | 文本 | 发起申请的用户姓名 |
| 申请时间 | 日期时间 | 申请提交时间 |
| 超期标识 | 标签 | 红色"超期"标签 |
| 待办数量 | 数字 | Tab标签旁显示数量角标 |

#### 5. 业务规则与约束
- **我的待办**：显示待当前用户审批的申请
- **我的申请**：显示当前用户发起的申请及审批进度
- **我的收藏**：显示当前用户收藏的数据资源
- **已办结**：显示当前用户已处理的审批事项
- 超期判定：申请提交超过3个工作日未处理标记为超期
- 审批超时处理：仅发送提醒通知，不自动通过或拒绝
- 默认显示最近5条记录

#### 6. 异常/边界场景
- 无待办事项时显示"暂无待办事项"
- 点击已过期权限申请提示"权限已过期，请重新申请"

### M3：核心数据概览模块

#### 1. 模块定义
展示平台核心数据资产统计概览，包括数据表、API、指标、报表四类资源的总量统计。

#### 2. 页面入口与路径
- **入口**：首页三列布局中上
- **路径**：首页 → 核心数据概览模块

#### 3. 核心交互流程
1. 用户查看四项核心指标
2. 点击指标卡片跳转至对应资源列表页
3. 点击"查看全部"跳转至资源目录页

#### 4. 字段/组件说明
| 组件名称 | 类型 | 说明 |
|---------|------|------|
| 数据表总量 | 整数 | 平台已注册的数据表数量 |
| API接口总量 | 整数 | 平台已发布的API接口数量 |
| 业务指标总量 | 整数 | 平台已定义的业务指标数量 |
| 报表总量 | 整数 | 平台已配置的报表数量 |

#### 5. 业务规则与约束
- 四项指标均为累计值（自平台上线以来）
- 仅统计已上架、未下架的资源
- 点击"查看全部"跳转至资源目录页

#### 6. 异常/边界场景
- 无数据时显示"暂无数据"
- 数据加载失败时显示错误提示

### M4：数据质量模块

#### 1. 模块定义
展示各部门数据质量达标情况，包括完整性、准确性、及时性三个维度的评分和综合达标率。

#### 2. 页面入口与路径
- **入口**：首页三列布局下方
- **路径**：首页 → 数据质量模块

#### 3. 核心交互流程
1. 用户查看各部门质量卡片
2. 点击"查看更多"跳转至数据质量详情页
3. 在详情页查看各部门详细质量数据

#### 4. 字段/组件说明
| 组件名称 | 类型 | 说明 |
|---------|------|------|
| 部门名称 | 文本 | 数据所属部门名称 |
| 达标状态 | 标签 | 达标/预警/未达标 |
| 数据资源数量 | 整数 | 该部门的数据资源项数 |
| 完整性得分 | 百分比 | 数据完整性评分 |
| 准确性得分 | 百分比 | 数据准确性评分 |
| 及时性得分 | 百分比 | 数据及时性评分 |
| 综合达标率 | 百分比 | 三项指标的加权综合得分 |

#### 5. 业务规则与约束
- 达标状态判定规则：
  - **达标**：综合达标率 ≥ 90%
  - **预警**：80% ≤ 综合达标率 < 90%
  - **未达标**：综合达标率 < 80%
- 颜色区分：
  - 绿色：达标
  - 蓝色：预警
  - 红色：未达标

#### 6. 异常/边界场景
- 无部门数据时显示空状态
- 点击卡片可跳转至该部门详情

### M5：资源推荐模块

#### 1. 模块定义
展示平台数据资源列表，支持按分类筛选、排序，用户可申请权限或收藏资源。

#### 2. 页面入口与路径
- **入口**：首页下方主区域
- **路径**：首页 → 资源推荐模块

#### 3. 核心交互流程
1. 用户点击分类标签筛选资源
2. 点击表头排序（热度/上架时间）
3. 点击资源名称查看详情
4. 点击"申请"按钮发起权限申请
5. 点击"收藏"按钮收藏资源

#### 4. 字段/组件说明
| 字段名称 | 类型 | 说明 |
|---------|------|------|
| 资源名称 | 文本 | 数据资源名称 |
| 资源类型 | 标签 | 数据表/报表/API/指标 |
| 所属分类 | 文本 | 生产运营类/安全环保类/人力资源类/供应链与物资类等 |
| 热度 | 整数 | 资源访问/申请次数 |
| 上架时间 | 日期 | 资源发布日期 |
| 状态 | 标签 | 可申请/审批中/已授权/已过期 |
| 操作按钮 | 按钮 | 收藏/申请 |

#### 5. 业务规则与约束
- 分类筛选逻辑：
  - 全部：显示所有资源
  - 按业务域分类：仅显示该分类下资源
- 排序规则：
  - 热度降序：访问量高的排前面
  - 时间降序：最新上架的排前面
- 状态说明：
  - 可申请：资源可正常申请
  - 审批中：用户已提交申请，等待审批
  - 已授权：用户已获得权限
  - 已过期：用户权限已过期

#### 6. 异常/边界场景
- 无资源时显示空列表
- 申请已授权资源提示"您已获得该资源权限"
- 申请已过期资源提示"权限已过期，请续期申请"

#### 2. 页面入口与路径
- **入口**：data-quality.html
- **路径**：首页 → 数据质量模块 → 查看更多 → 部门质量概览

#### 3. 核心交互流程
1. 用户查看各部门质量卡片列表
2. 点击部门卡片查看该部门详细质量数据
3. 点击"返回"返回首页

#### 4. 字段/组件说明
| 字段名称 | 类型 | 说明 |
|---------|------|------|
| 部门名称 | 文本 | 数据所属部门名称 |
| 达标状态 | 标签 | 达标/预警/未达标 |
| 数据资源数量 | 整数 | 该部门的数据资源项数 |
| 完整性得分 | 百分比 | 数据完整性评分 |
| 准确性得分 | 百分比 | 数据准确性评分 |
| 及时性得分 | 百分比 | 数据及时性评分 |
| 综合达标率 | 百分比 | 三项指标的加权综合得分 |
| 更新时间 | 日期时间 | 质量数据最后更新时间 |

#### 5. 业务规则与约束
- 达标状态判定规则：
  - **达标**：综合达标率 ≥ 90%（绿色）
  - **预警**：80% ≤ 综合达标率 < 90%（蓝色）
  - **未达标**：综合达标率 < 80%（红色）
- 综合达标率计算：完整性×40% + 准确性×35% + 及时性×25%
- 按综合达标率升序排列（未达标排前面）

#### 6. 异常/边界场景
- 无部门数据时显示空状态
- 数据加载失败时显示错误提示

- 已授权资源显示"已授权"状态，不可重复申请
- 审批中资源显示"审批中"状态，不可重复申请
- 已过期资源显示"已过期"状态，可续期申请
- 收藏状态即时更新

#### 6. 异常/边界场景
- 资源已下架时提示"该资源已下架"
- 无权限查看详情时提示"您无权查看该资源详情"

### M12：权限申请流程模块

#### 1. 模块定义
用户申请数据资源访问权限的流程模块，包括申请表单填写、提交、审批进度查看。

#### 2. 页面入口与路径
- **入口**：资源详情弹窗 → 申请权限按钮
- **路径**：资源详情 → 申请权限 → 填写申请表单 → 提交申请

#### 3. 核心交互流程
1. 用户点击"申请权限"按钮
2. 填写申请表单（申请原因、使用期限等）
3. 提交申请
4. 系统自动分配审批人
5. 用户可在"我的申请"中查看审批进度

#### 4. 字段/组件说明
| 字段名称 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| 资源名称 | 文本 | 自动 | 自动填充，不可修改 |
| 申请人 | 文本 | 自动 | 当前登录用户，自动填充 |
| 申请部门 | 文本 | 自动 | 当前用户所属部门，自动填充 |
| 申请原因 | 文本域 | 是 | 申请使用该资源的原因，最多500字 |
| 使用期限 | 下拉选择 | 是 | 三个月/六个月/一年/永久 |
| 提交按钮 | 按钮 | - | 提交申请 |
| 取消按钮 | 按钮 | - | 取消申请 |

#### 5. 业务规则与约束
- **权限有效期**：默认三个月，到期需续期申请
- **到期提醒**：权限到期前7天发送提醒通知
- **到期处理**：权限到期后自动失效，需重新申请
- **审批超时**：超过3个工作日未审批仅发送提醒，不自动处理
- **重复申请**：审批中或已授权状态不可重复申请
- **申请被拒**：被拒绝后可重新申请，无时间限制

#### 6. 异常/边界场景
- 网络异常时提示"提交失败，请重试"
- 必填项未填写时提示"请填写完整信息"
- 申请原因字数超限时提示"申请原因不能超过500字"

| 权限模型 | RBAC（基于角色的访问控制） |
| 接口鉴权 | JWT Token验证 |

### 14.2 数据安全

| 安全项 | 说明 |
|-------|------|

## 十六、业务规则补充

### 16.1 审批人分配规则

| 场景 | 审批人 | 说明 |
|------|-------|------|
| 数据表权限申请 | 资源所属部门的部门主管 | 系统自动分配 |
| API权限申请 | 资源所属部门的部门主管 | 系统自动分配 |
| 报表权限申请 | 资源所属部门的部门主管 | 系统自动分配 |
| 指标权限申请 | 资源所属部门的部门主管 | 系统自动分配 |

### 16.2 资源责任人管理

| 规则项 | 说明 |
|-------|------|
| 责任人数量 | 每个资源支持设置1名主责任人和多名协作者 |
| 责任人变更 | 需要数据管理员审批后生效 |
| 责任人权限 | 责任人可编辑资源信息、查看使用统计 |

### 16.3 部门数据归属

| 场景 | 归属规则 |
|------|---------|
| 单部门数据 | 归属该部门 |
| 多部门共享数据 | 归属数据生产部门（数据源头部门） |
| 跨部门数据 | 归属数据治理部门或指定部门 |

### 16.4 权限回收机制

| 场景 | 处理方式 |
|------|---------|
| 用户离职 | HR系统同步离职状态后，自动回收所有权限 |
| 用户调岗 | 保留历史权限，新岗位权限需重新申请 |
| 权限到期 | 到期后自动失效，系统记录失效日志 |
| 资源下架 | 自动回收所有用户对该资源的权限 |

### 16.5 资源类型扩展

系统预留资源类型扩展能力，未来可支持：
- 数据服务（Data Service）
- 数据应用（Data App）
- 数据模型（Data Model）
- 数据标签（Data Tag）

扩展时只需在资源类型枚举中新增类型，无需修改核心代码结构。
